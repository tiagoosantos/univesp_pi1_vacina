Compreendo o seu pedido! Para criar um site que permite consultar vacinas a partir de nome e CPF, utilizando um banco de dados fictício, precisaremos de algumas tecnologias. Vou te apresentar um plano e um código de exemplo, mas é importante lembrar que um site completo e seguro envolve mais etapas e conhecimentos.

Tecnologias Necessárias:

  • HTML: Para a estrutura da página web (formulário, exibição de resultados).
  • CSS: Para estilizar a página (deixar bonita e organizada).
  • JavaScript: Para a lógica do lado do cliente (pegar os dados do formulário, "consultar" o banco de dados fictício e exibir os resultados).
  • NÃO usaremos um banco de dados real: Como você pediu um banco de dados fictício, simularemos isso diretamente no código JavaScript, usando um array de objetos. Para um projeto real, você usaria tecnologias como Python/Django, Node.js/Express, PHP/Laravel, ou Ruby on Rails com bancos de dados como PostgreSQL, MySQL ou MongoDB.

Estrutura do Projeto (Arquivos):

  1. index.html (o arquivo principal da página)
  2. style.css (para o estilo)
  3. script.js (para a lógica JavaScript)

Passo a Passo da Programação:

1. index.html (Estrutura da Página)

Este arquivo conterá o formulário para entrada de nome e CPF, e a área onde os resultados serão exibidos.

<!DOCTYPE html>

</html>

3. script.js (Lógica e Banco de Dados Fictício)

Este é o coração da aplicação. Ele conterá o "banco de dados" de pessoas e vacinas, e a lógica para buscar e exibir os resultados.

JavaScript

document.addEventListener('DOMContentLoaded', () => {
   const consultaForm = document.getElementById('consultaForm');
   const resultadoDiv = document.getElementById('resultado');

   // **Banco de Dados Fictício**
   // Em um cenário real, isso viria de um backend/API que se conecta a um banco de dados.
   const dadosVacinasFicticios = [
       {
           nome: "João Silva",
           cpf: "12345678901",
           vacinas: [
               { nome: "Tétano", data: "01/03/2020" },
               { nome: "Hepatite B", data: "15/07/2019" },
               { nome: "Febre Amarela", data: "10/01/2022" }
           ]
       },
       {
           nome: "Maria Oliveira",
           cpf: "98765432109",
           vacinas: [
               { nome: "Gripe (Influenza)", data: "05/05/2023" },
               { nome: "COVID-19 (1ª Dose)", data: "20/08/2021" },
               { nome: "COVID-19 (2ª Dose)", data: "20/09/2021" },
               { nome: "COVID-19 (Reforço)", data: "10/04/2022" }
           ]
       },
       {
           nome: "Pedro Souza",
           cpf: "11223344556",
           vacinas: [
               { nome: "Tríplice Viral", data: "22/11/2018" },
               { nome: "DTP (Difteria, Tétano, Coqueluche)", data: "12/06/2017" }
           ]
       },
       {
           nome: "Ana Costa",
           cpf: "00112233445",
           vacinas: [
               { nome: "HPV (1ª Dose)", data: "03/02/2023" }
           ]
       }
   ];

   consultaForm.addEventListener('submit', (event) => {
       event.preventDefault(); // Impede o envio padrão do formulário

       const nomeInput = document.getElementById('nome').value.trim().toLowerCase();
       const cpfInput = document.getElementById('cpf').value.trim();

       // Limpa resultados anteriores
       resultadoDiv.innerHTML = '';

       // Validação básica de CPF (apenas números e 11 dígitos)
       if (!/^\d{11}$/.test(cpfInput)) {
           resultadoDiv.innerHTML = '<p class="mensagem-erro">Por favor, digite um CPF válido (apenas 11 números).</p>';
           return;
       }

       const pessoaEncontrada = dadosVacinasFicticios.find(pessoa =>
           pessoa.cpf === cpfInput && pessoa.nome.toLowerCase().includes(nomeInput)
       );

       if (pessoaEncontrada) {
           let htmlResultado = `<h2>Vacinas de ${pessoaEncontrada.nome}</h2>`;
           if (pessoaEncontrada.vacinas.length > 0) {
               htmlResultado += '<ul>';
               pessoaEncontrada.vacinas.forEach(vacina => {
                   htmlResultado += `<li><strong>${vacina.nome}</strong> - Data: ${vacina.data}</li>`;
               });
               htmlResultado += '</ul>';
           } else {
               htmlResultado += '<p>Nenhuma vacina encontrada para este CPF e nome.</p>';
           }
           resultadoDiv.innerHTML = htmlResultado;
       } else {
           resultadoDiv.innerHTML = '<p class="mensagem-erro">Pessoa não encontrada ou dados incorretos. Verifique o nome e o CPF.</p>';
       }
   });
});

Como executar o site:

  1. Crie uma pasta no seu computador (ex: site_vacinas).
  2. Dentro dessa pasta, crie os três arquivos: index.html, style.css e script.js.
  3. Copie e cole o código correspondente em cada arquivo.
  4. Abra o arquivo index.html no seu navegador web (clique duas vezes nele ou arraste-o para a janela do navegador).

Explicação e Considerações Importantes:

  • Banco de Dados Fictício: O array dadosVacinasFicticios no script.js atua como seu "banco de dados". Em um projeto real, esses dados seriam buscados de um servidor (backend) que se conecta a um banco de dados de verdade.
  • Busca: A função find() do JavaScript é usada para procurar uma pessoa que corresponda ao CPF e que o nome digitado esteja contido no nome completo cadastrado (assim, "João" encontra "João Silva").
  • Segurança: Este exemplo é apenas para fins ilustrativos. Em um sistema real:
  • Você NUNCA deve armazenar dados sensíveis como CPF diretamente no frontend ou em arquivos JavaScript.
  • A validação de CPF deve ser muito mais robusta e preferencialmente feita no servidor.
  • As consultas a um banco de dados seriam feitas através de APIs seguras (endpoints) que o JavaScript do frontend chamaria.
  • Seriam necessárias autenticação e autorização para garantir que apenas pessoas autorizadas acessem os dados.
  • Design Responsivo: O CSS fornecido é básico. Para um site que se adapta bem a diferentes tamanhos de tela (celulares, tablets, desktops), você precisaria de um CSS mais avançado (media queries).
  • Manipulação de Erros: O tratamento de erros é rudimentar. Em um sistema real, você teria mensagens de erro mais detalhadas e um registro de logs.

Este projeto te dará uma base funcional para entender como um site desse tipo poderia operar no lado do cliente. Para ir além e criar algo robusto e seguro, o próximo passo seria aprender sobre desenvolvimento de backend e bancos de dados.
